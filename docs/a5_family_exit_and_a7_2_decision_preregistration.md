## Candidate A7.2 Mechanism

A7.2, if Ben accepts it, should be a new preregistered single-hive gate rather
than another A5 seed broadening. The mechanism should make prediction
endogenous and delayed:

- agents choose among `predict`, `work`, `review`, and `synthesize`;
- utilities are thresholded logistic functions of lagged forecast error,
  artifact readiness, contradiction, risk, fatigue, and adaptive thresholds;
- prediction consumes work opportunity and can update only lagged or delayed
  forecast/artifact state;
- review and synthesis depend on artifact readiness and contradiction/risk,
  not only queue depth;
- fatigue and adaptive thresholds create bounded hysteresis, with all slopes,
  delays, costs, and caps frozen before any run.

The scientific question would be whether delayed, artifact-mediated,
resource-bounded endogenous prediction produces residual structure that
survives accounting and null controls, not whether it merely improves forecast
skill or changes backlog.

## Required Endpoints

Primary endpoints should be frozen before implementation:

- forecast skill per prediction spend;
- residual predictability after accounting for clock/demand phase, task
  arrivals, service capacity, action opportunity, work budget, prediction
  spend, remaining work, backlog, queued age, completion fraction, starvation,
  and source-ledger components;
- lead-lag structure from lagged error/uncertainty to prediction spend and
  artifact state;
- delay-embedded recurrence or return structure in residual artifact/prediction
  state;
- macro-state compressibility or predictability that is not reducible to
  queue/accounting variables;
- productivity guardrails for completion fraction, backlog, queued age,
  starvation, prediction-spend volatility, and work-budget volatility.

## Required Nulls And Controls

A7.2 should not run without prospectively frozen controls:

- zero-budget reactive baseline;
- high-budget or oracle smoothing positive control;
- amplitude-matched linear control;
- same-tick logistic control;
- phase-shuffled lag-input control;
- threshold-shuffled control;
- source-preserving semantic label shuffle;
- spend-only replay null that deducts the same prediction/work opportunities
  while removing useful forecast timing.

Promotion would require an intermediate endogenous-prediction condition to beat
the relevant controls on preregistered residual endpoints while preserving
guardrails. Forecast skill alone is insufficient.

## Closure Rules

Close A7.2 fail-closed if residual endpoints are explained by accounting
controls, matched nulls, source-ledger components, backlog dwell, or work-loss
mediation. Do not rescue a negative result by tuning slopes, thresholds,
delays, costs, or caps after seeing residual plots.

If Ben prefers multi-hive work, do not implement it from this document. Write a
separate three-hive ring preregistration with cross-hive prediction costs,
delayed artifact transfer, target/phase nulls, and transfer-opportunity
